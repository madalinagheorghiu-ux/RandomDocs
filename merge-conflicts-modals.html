Purpose
Shown before conflict resolution begins when resources in the two branches share the same unique identifiers. Advises the user to create new branch versions and rename identifiers before proceeding. Does not allow proceeding — only closing.

Merge Conflicts List

flx-merge-conflicts-list
File
…/merge-conflicts/components/merge-conflicts-list/merge-conflicts-list.component.ts
Route
/:mergeConflictsId (default child)
Purpose
Overview of all detected conflicts, grouped by resource type in collapsible accordions. User can navigate to each conflict to resolve it, or trigger a merge (with or without reviewing everything).
Content
  • Branch metadata header (merge-from / merge-over branch names)
  • Tip: changes from the source branch are prioritized unless overridden
  • Global conflicts (metadata, config parameters, etc.) as individual links
  • Resource conflicts (Processes, Views, Enumerations, etc.) in accordions with counts
  • Each row links to the detailed conflict viewer
Actions
  • Discard Merge – navigates to discard-confirmation
  • Merge – triggers merge; if unreviewed items remain, navigates to confirmation; otherwise merges directly. Disabled when there are unresolved errors.

File

Used by
flx-conflict-viewer when the conflict is NOT a dependency conflict
Purpose
3-pane CodeMirror merge editor. Left = source branch, right = target branch, center = editable result. Supports scripting language mismatch detection with per-side "select all" buttons.
Special states
  • Different scripting languages – shows language pills and "select all from left/right" bulk-accept buttons
  • Parent deleted – editors are disabled with an orange warning: "Parent conflict resolved as 'deleted'…"
  • Invalid code – footer shows orange warning: "Error invalid code. Changes will not be saved."
